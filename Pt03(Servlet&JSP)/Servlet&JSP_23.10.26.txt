1. 웹 프로그램의 3단계
- 클라이언트측, 웹서버측, db측
2. 웹서버측의 구성: 웹서버 + WAS, 2개 각각의 역할
- 웹서버(http요청/응답), WAS(자바로 동적처리부분담당)
3. WAS는 무엇의 약자?
- Web Application Server
4. 웹서버 종류?
- Apache, nginx(엔진엑스), IIS
5. 어떤 자바 WAS를 써봤나요?
- Tomcat
6. JDBC 4단계를 정리해 보세요
- 드라이버 설정, DB연결, sql문 생성, sql문 db로 전송
7. mysql과 oracle의 기본 port
- 3306, 1521
8. JSP는 무엇의 약자?
- Java Server Pages
9. servlet과 JSP의 관계?
- JSP는 서블릿이라는 자바프로그램으로 변환되어 실행, service()
10. http의 용도, 무엇의 약자?
- 클라이언트와 웹서버의 통신규칙(규약, 프로토콜), HyperText Transfer Protocol
11. http의 구성요소 2가지
- header + body
12. 브라우저가 보낼 때의 http 헤어와 바디
- header(요청주소 + get(데이터)), body(post(데이터))
13. 웹서버가 보낼 때의 http헤더와 바디
- header(상태코드), body(처리결과)
